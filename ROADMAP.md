# Zorya — план розвитку та виходу в продакшен

> Оновлено: 2026-07-15
> Стан на момент написання: робочий прототип — auth із refresh-flow, рушій на Swiss Ephemeris
> (натал / транзити / соляр / синастрія), атлас на 34k міст, арабські частини, терми,
> SVG-колесо з тултіпами й градусами куспідів, модуль персон, денний ліміт free-плану,
> Stripe/LiqPay підключені, але з відомими багами. 32 тести проходять.
> Історичний аналіз ZET9 і початковий план — див. [PLAN.md](PLAN.md).

---

## ⚠️ Блокери, які треба вирішити ДО прийому грошей

### 1. Ліцензія Swiss Ephemeris — ✅ РІШЕННЯ: власний рушій
`pyswisseph` — AGPL: для закритого SaaS довелося б або відкривати код бекенду,
або купувати комерційну ліцензію Astrodienst (~CHF 750).

**Обрано третій шлях — власний рушій без ліцензійних зобов'язань:**
- Позиції планет: **Skyfield (MIT)** + ефемериди **JPL DE440s (суспільне надбання)** —
  ті самі сирі дані, які Swiss Ephemeris лише стискає; точність та сама або краща
- Будинки: власна реалізація (Placidus — ітерація за умовою `RA(λ)=RAMC+offset+f·AD(λ)`,
  Regiomontanus/Campanus — точна векторна геометрія, Koch, Equal, Whole Sign)
- Вузли: істинний — з оскулюючих елементів орбіти Місяця; середній вузол і Ліліт —
  аналітичні поліноми (Meeus, суспільне надбання)
- Хірон: опційний SPK-файл із JPL Horizons (безкоштовний)
- **pyswisseph лишається dev-залежністю як еталон**: у CI кожна зміна рушія
  крос-валідується проти Swiss Ephemeris із секундною точністю (AGPL не поширюється
  на внутрішнє тестування — код не обслуговує користувачів)

### 2. Обіцянки Pricing-сторінки
Pro-план зараз обіцяє **PDF export, custom orbs, progressions** — жодного немає в коді.
До першого платежу: реалізувати або прибрати з опису тарифу.

### 3. Назва
«Zorya» обрана як робоча, але домен і торгова марка **не перевірені**.
Перевірити zorya.app / zorya.com.ua та збіги в App Store перед брендингом.

## Етап B — Інфраструктура (тижні 2–3): продакшен-середовище

Реалізовано 2026-07-16, план: [docs/plans/2026-07-15-stage-b-infrastructure.md](docs/plans/2026-07-15-stage-b-infrastructure.md).
Docker на dev-машині відсутній — усе, що потребує живого Postgres, верифікується
через новий CI job `backend-postgres` (GitHub Actions service container); саму
збірку образів і `docker compose up` ще належить прогнати на реальному VPS
(DoD 5 плану лишається відкритим до першого деплою).

- [x] **PostgreSQL** замість SQLite: asyncpg, dual-dialect підтримка (моделі без
      змін); пошук міст — `pg_trgm` (GIN + similarity) на Postgres, SQLite FTS5
      без змін лишається для dev
- [x] **Хостинг**: `infra/docker-compose.yml` (db + api + Caddy) готовий до
      Hetzner VPS; фронтенд на Vercel — розгортання ще не виконане (ручний крок,
      див. docs/DEPLOY.md)
- [ ] **Домен**: перевірити та купити (див. блокер №3) — не зроблено, ручна дія
- [x] **Операційка**:
  - [x] Sentry: backend gated на `SENTRY_DSN`, frontend (instrumentation.ts +
        instrumentation-client.ts) gated на `NEXT_PUBLIC_SENTRY_DSN` — обидва
        no-op, поки DSN не задано
  - [ ] Uptime-моніторинг — задокументовано в docs/DEPLOY.md, не активовано
        (потребує реального домену)
  - [x] `scripts/backup_db.sh`: pg_dump + gzip + ротація 14 днів, задокументовано
        в docs/DEPLOY.md; реальний прогін на проді — після деплою
  - [x] Секрети через env; fail-fast на дефолтний `SECRET_KEY` (з Етапу A)

## Етап D — Запуск (тиждень 5)

В процесі, плани: [docs/plans/2026-07-16-stage-d-launch.md](docs/plans/2026-07-16-stage-d-launch.md),
[docs/plans/2026-07-18-monopay-migration.md](docs/plans/2026-07-18-monopay-migration.md).
Ручні кроки власника перед публічним запуском (домен, live-ключі, деплой,
юридичне рев'ю, бета) — окремий чекліст: [docs/LAUNCH.md](docs/LAUNCH.md).

- [x] Чесні тарифи: Pro без нереалізованих фіч, Expert прибрано з продажу
- [x] Бекенд-харденінг: rate limits на reset/verify-токени, bcrypt-ліміт
      пароля, dependency audit (точкові фікси, без масових бампів)
- [x] Security-заголовки фронтенда (мінімальний CSP — без script/style/img-src,
      щоб не зламати Next hydration і OSM-тайли)
- [x] Security-рев'ю всієї гілки + виправлення знахідок — план **C3**,
      реалізовано 2026-07-19:
      [docs/plans/2026-07-19-c3-security-review.md](docs/plans/2026-07-19-c3-security-review.md).
      7 знахідок (усі 8 роутерів, обидва платіжні вебхуки, ін'єкції,
      секрети/PII/CORS): 5 виправлено з регрес-тестами, 2 — свідомо
      прийнятий ризик з обґрунтуванням. Найважливіші фікси: monopay
      webhook міг реактивувати скасовану підписку через replay (High);
      chart-ендпоінти взагалі не мали rate limit (Medium, DoS); atlas
      FTS5-пошук ламався на реальних назвах міст з апострофом/дефісом
      (Medium/функціонально-High); login мав timing-side-channel;
      email-токени мали TOCTOU-гонку. Точковий CVE-фікс:
      `cryptography`→48.0.1, `pytest`→9.1.1.
- [x] E2E-тест оплати: Stripe test mode повний цикл (оплата → webhook →
      план → скасування → downgrade) живою карткою 4242…; LiqPay замінено
      на **monopay** (еквайринг monobank) — checkout/redirect/sync/cancel
      перевірено живцем, але повний цикл "оплата картою → success-вебхук"
      НЕ пройдено живою карткою (немає задокументованих monobank
      test-номерів, реальною карткою ризикувати не стали) — ця гілка
      верифікована лише юніт-тестами; обов'язковий ручний прогін живою
      оплатою перед публічним запуском — див. docs/LAUNCH.md
- [x] Plausible analytics (GDPR-friendly, cookie-less, без cookie-банера) —
      gated на `NEXT_PUBLIC_PLAUSIBLE_DOMAIN`, без нього — нуль запитів
- [x] Feedback-лінк у дашборді, gated на `NEXT_PUBLIC_FEEDBACK_EMAIL`
- [ ] Закрита бета на 10–20 користувачів — насамперед астрологи-практики:
      вони одразу зловлять розбіжності точності, а точність — головна цінність продукту
- [ ] Публічний запуск
